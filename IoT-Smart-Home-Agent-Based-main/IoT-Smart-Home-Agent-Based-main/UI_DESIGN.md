# Mobile App UI Design - IoT Smart Home Agent

## Overview

The mobile application is built using **React Native** with **Expo** framework, providing a cross-platform solution for both iOS and Android devices.

---

## Technology Stack

| Technology | Purpose |
|------------|---------|
| **React Native** | Cross-platform mobile framework |
| **Expo** | Development toolchain & APIs |
| **TypeScript** | Type-safe JavaScript |
| **Zustand** | State management |
| **React Query** | Server state & caching |
| **Expo Audio** | Voice recording |
| **WebSocket** | Real-time communication |

## Design Patterns Used

### 1. Card-Based Layout
All components use card containers with:
- White background
- Rounded corners (12px)
- Shadow for elevation
- Consistent padding (16px)

### 2. Grid System
- 2-column grid for sensor data
- Full-width cards for controls
- Responsive spacing

### 3. State Indicators
- Color-coded status (green/red)
- Icon + text combinations
- Loading states for async operations

### 4. Touch Feedback
- Button press animations
- Haptic feedback on actions
- Disabled state styling

## Accessibility Features

| Feature | Implementation |
|---------|---------------|
| **Large Touch Targets** | Minimum 44x44px for buttons |
| **High Contrast** | Dark text on light backgrounds |
| **Status Indicators** | Color + text (not just color) |
| **Screen Reader** | Semantic labels on all components |
| **Error Messages** | Clear, descriptive alert text |

## State Management (Zustand)

```typescript
interface AppState {
  // Telemetry state
  telemetry: TelemetryData | null;
  setTelemetry: (data: TelemetryData) => void;

  // Connection state
  isConnected: boolean;
  setIsConnected: (connected: boolean) => void;

  // Voice command state
  isRecording: boolean;
  setIsRecording: (recording: boolean) => void;

  lastCommandResponse: string;
  setLastCommandResponse: (response: string) => void;

  // UI state
  showEmergencyAlert: boolean;
  setShowEmergencyAlert: (show: boolean) => void;
}
```
